
Practical Task

Create a Private Cloud-Based Document Sharing Platform

• Private document storage
  → Users can upload files like PDF, DOCX, XLSX, PPT, etc.
  → Files are stored securely in cloud storage (for now local server).
  → Nothing is publicly accessible.

• Strict access control
  → No public share links.
  → A file is visible only to the owner by default.
  → The owner must manually invite specific users.
  → Only those selected users can view/download the file.
  → Any other user must be completely denied access.

• Simple authentication system
  ✓ Only email + password login (no social login, no OAuth).
  ✓ Users can register.
  ✓ Password reset and change password features are required.
  → Basic profile management.

• Admin panel
  ✓ Admin can see all users.
  → Admin can see total storage usage.
  → Admin can instantly grant or revoke access.
  → Admin credentials must be provided.

• Responsive UI
  → Must work properly on mobile and desktop.
  → Clean web interface.


1. GUEST (Unauthenticated User)

This is someone who has not logged in yet.

Modules:
✓ Registration (Email + Password)
✓ Login
✓ Forgot Password
✓ Reset Password
✓ Email verification (recommended for security)

Access Restrictions:
→ Cannot view any documents
→ Cannot see any users
→ Cannot access any file URL directly


2. REGISTERED USER (Document Owner / Shared User)

This is the main system user.

A) Account Management
→ Update profile (name, email)
✓ Change password
→ View storage usage (optional but recommended)

B) Document Management (Owner Side)
→ Upload document (PDF, DOCX, XLSX, PPT, etc.)
→ Store in object storage (local)
→ Create folders (optional but useful)
→ Rename file
→ Delete file
→ Download file
→ View file details (size, upload date, shared users)

C) Invite-Only Sharing System
→ Share file with specific registered users only
→ Select user(s) from existing users
→ Revoke access anytime
→ View list of users who have access

Important logic:
→ By default, file visibility = Private
→ Only owner + explicitly invited users can access
→ No public links
→ Every file access must validate permissions from DB

D) Shared With Me Section
→ List of files shared by others
→ Download allowed files
→ No ability to reshare unless permission is granted


3. ADMIN ROLE

This is system-level management.

A) User Management
✓ View all registered users
✓ Activate / deactivate users
✓ Delete users
✓ Reset user passwords (optional)

B) Storage Monitoring
→ View total system storage usage
→ View per-user storage usage
→ Identify heavy storage users

C) Access Control Override
→ Grant or revoke document access manually
→ Force delete any document
→ Disable sharing for specific users

D) System Monitoring

Audit logs:
• Who uploaded what
• Who accessed which file
• Who shared with whom

Basic dashboard:
• Total users
• Total documents
• Total storage used


4. SYSTEM MODULES (Common Layer)

→ Authentication module
→ Role & permission system
→ File upload service
→ Object storage integration
✓ Access validation middleware
→ Email service (for invite notification + reset password)
→ API layer (if frontend is separated)
→ Docker configuration
✓ Environment config (.env management)


5. DATABASE CORE TABLES (High-Level)

✓ users
✓ documents
→ folders (optional)
✓ document_user_permissions (pivot table)
→ storage_logs
✓ password_resets
✓ audit_logs
